全库健康检查(每月或大改后执行):

0. **检查点**:`git add -A && git commit -m "chore: lint 前检查点"`。
1. **断链**:解析全库 `[[wikilink]]`(含 `![[嵌入]]`),列出指向不存在文件的链接。可写脚本批量扫(Obsidian 短名解析:文件名在全库唯一即命中)。
2. **孤页**:`30.Resources/` 中无任何入链、也不在任何索引里的页 → 加 `#codex/orphan`。
3. **frontmatter**:YAML 不可解析、或 wiki 页缺 `type/title` 的文件清单。
4. **陈旧**:`confidence < 0.4` 或 90 天未更新的重要页 → 加 `#codex/stale`。
5. **矛盾**:同一事实在两页描述不一致 → 两页都加 `#codex/conflict`,**不自动裁决**,列出待馆主复核。
6. **索引一致性**:`_index.md` 的分类计数与实际文件数核对,过时条目修正。
7. **修复与报告**:断链和索引可直接修;事实性改写需确认。`_log.md` 顶部追加 `## [YYYY-MM-DD] lint | 结果摘要`,输出体检报告(各项数量 + 待人工处理清单)。
