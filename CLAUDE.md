## 二、三层四环架构

**三层(存什么)**

| 层 | 位置 | 规则 |
|----|------|------|
| L0 原始层 | `*原始资料/`、`Clippings/`、PDF | 不可变,带 SHA-256,AI 永不改写 |
| L1 编译层 | `30.Resources/` wiki 页、`10.Projects/`、`20.Areas/`、`50.Daily/` | AI 编译维护,持续生长 |
| L2 系统层 | `CLAUDE.md`、`memory.md`、`.claude/commands/`、git | 管运行,改动需走红线流程 |

**四环(怎么转)**:捕捉(人丢 Inbox)→ 入库 `/ingest` → 调用 `/ask` `/daily` → 进化 `/crystallize` `/lint` `/review`

**目录**

```
00.Inbox/     捕捉入口,整理后清空      30.Resources/  Codex Wiki + 原始资料
10.Projects/  进行中项目               40.Archive/    已完成/暂停
20.Areas/     持续关注领域             50.Daily/      每日笔记
Clippings/    网页剪藏落点(待入库)     Templates/     模板 ｜ docs/ 系统文档
```

**晋升链**:Daily 反复出现的想法 → 提炼进 `30.Resources/`;项目结束 → 结晶关键知识后整体归档 `40.Archive/`。

## 三、检索协议 ⭐(每次回答前的标准动作)

**先查库,再生成。**

```
① 查索引  → [[30.Resources/_index]] 及子索引定位候选笔记(偏好类问题查 memory.md)
② 读原文  → 打开命中笔记读实际内容,不靠记忆
③ 提炼输出 → 基于读到的内容综合回答,标注来源 wikilink
④ 归档新知 → 有价值的新合成 → 存为 wiki 页(/ask 自动做)
⑤ 诚实兜底 → 库中无相关内容则明说「知识空白」,记入 _index 待补建;大模型补充要声明非来自库
```

## 四、写入规范

**wiki 页元数据(必填项)**

```yaml
type: codex-wiki
title: 标题
epistemic_status: fact | verified_inference | hypothesis | decision | superseded
created: YYYY-MM-DD
updated: YYYY-MM-DD
source: 来源(重要结论必填)
tags: [topic/xxx]
```

**规则**

- 重要结论必须有 `source`;精确数字/公式/题目答案回原始层核对
- 新旧冲突:标 `#codex/conflict`,复核适用条件,**不自动认定新的正确**
- 替代:旧页加 `superseded_by: "[[新笔记]]"`,不删原文
- 无来源证据的关系只用普通 wikilink,不写成因果/依赖
- 不自动衰减 confidence、不自动裁决冲突、不提前部署向量库(保守边界)
- 每次库级操作(ingest/store/lint/crystallize)完成后在 [[30.Resources/_log]] 顶部追加记录

## 五、命令系统(流程的唯一权威定义在 `.claude/commands/`)

| 命令 | 何时用 | 做什么 |
|------|--------|--------|
| `/daily` | 每天第一次会话 | 建 daily、带入待办、扫 Inbox、载入 memory |
| `/ingest` | Inbox/Clippings 有料 | 编译入库:提炼→双链→索引→日志→清空 |
| `/ask` | 知识问答 | 执行检索协议,新知归档 |
| `/crystallize` | 会话出结论时(闭馆) | 结晶:问题/证据/结论/边界/被拒方案 |
| `/lint` | 每月或大改后 | 体检:断链/孤页/陈旧/矛盾/frontmatter |
| `/review` | 每周一次 | 复盘:晋升 Daily 想法、连接项目、清知识空白 |

## 六、记忆协议

[[memory.md]] = 本宪法的动态补丁,**冲突时 memory 优先**。四个触点:启动读取 → 检索查询 → 输出前校验 → 发现新偏好即写入(当天 daily 留痕)。只记你明确表达或反复验证的内容;过时条目标 `~~已过时~~` 不删。

## 七、行为准则与红线

- **语言**:【你的偏好语言】;严谨、基于事实;内容宁多勿少,**结构直观简洁**(不深嵌套、不造重复文档)
- **主动**:发现遗漏/错分类/断链主动提;信息不全主动问;不猜测不脑补
- **红线**:
  - 只用你亲口提供的信息构建画像
  - 批量/结构性改动:先 `git commit` 检查点再动手;未获授权的结构变更先说明理由等确认;已授权的长任务一次执行完毕后统一汇报
  - L0 原始层永不改写;删除类操作必须可回滚(git)
  - 系统文档单一真相源:完整规则只在本文件,`AGENTS.md` 只做指针

## 八、规模升级触发线
